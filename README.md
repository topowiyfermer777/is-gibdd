# Информационная система ГИБДД (ИС ГИБДД)

Веб-приложение для автоматизации процессов деятельности подразделений Государственной инспекции безопасности дорожного движения: регистрации ТС, учета ДТП, ведения технических осмотров и розыска угнанных транспортных средств.

---

## 🚀 Быстрый запуск с помощью Docker Compose

Для локального развертывания всей системы в изолированных контейнерах (бэкенд FastAPI + фронтенд React на Nginx + база данных SQLite с персистентным хранением) выполните одну команду из папки `/is`:

```bash
docker compose up --build -d
```

### Доступ к системе:
- **Веб-интерфейс (фронтенд):** [http://localhost:5177/](http://localhost:5177/)
- **REST API Swagger (бэкенд):** [http://localhost:8080/docs](http://localhost:8080/docs)

---

## 🔑 Учетные записи для тестирования (встроены при инициализации)

При первом запуске база данных автоматически создается и наполняется тестовыми пользователями. Выберите одну из ролей и войдите в систему с соответствующим табельным номером (пароль для всех: `password`):

| Табельный номер | Роль | Описание роли | Доступные экраны |
| :--- | :--- | :--- | :--- |
| **1001** | `admin` | Администратор системы | Доступ ко всем страницам, логам аудита и операциям |
| **1002** | `inspector` | Инспектор ДПС | Оформление ДТП, розыск ТС, проверка по базе (ПД замаскированы) |
| **1003** | `registrar` | Регистратор | Постановка ТС на учет, снятие с учета, выдача свободных ГРЗ |
| **1004** | `to_employee` | Сотрудник ТО | Внесение результатов техосмотра, просмотр реестра пунктов ТО |
| **1005** | `analyst` | Аналитик / Руководитель | Просмотр графиков угонов, дашборд статистики ТС, полный доступ к ПД |

## 🛠️ Структура проекта

- **`/backend`** — Серверная часть (FastAPI, SQLAlchemy, SQLite). Автоматически инициализирует базу данных `gibdd.db` и запускает скрипт наполнения при первом старте.
- **`/frontend`** — Клиентская часть (React, Vite, Tailwind CSS v4, Nginx). Собирает статику и раздает ее на порту `80` внутри контейнера (проброшен на хост-порт `5177`).

---

## 💾 Сохранность данных

Вся база данных хранится во встроенном томе Docker (`db-data`). При выполнении `docker compose down` и последующем перезапуске все ваши зарегистрированные ТС, записи о розыске и логах аудита **сохраняются**.

Для полного сброса базы данных и повторного наполнения тестовыми данными используйте команду:
```bash
docker compose down -v
```
