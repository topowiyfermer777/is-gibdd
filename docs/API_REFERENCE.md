# Спецификация REST API — Информационная система ГИБДД

Данный документ содержит подробное описание REST API Информационной системы ГИБДД (учебный проект), включая адреса эндпоинтов, структуры запросов и ответов, а также ролевую модель доступа.

---

## 🔑 1. Общие сведения и авторизация

- **Базовый URL:** По умолчанию API запускается на `http://localhost:8080` (или `http://127.0.0.1:8080`).
- **Формат данных:** Все запросы и ответы используют формат `application/json` (кроме авторизации, принимающей URL-encoded форму).
- **Схема авторизации:** Стандартный протокол OAuth2 с использованием Bearer JWT-токенов.
  - Для защищенных маршрутов необходимо передавать заголовок:
    `Authorization: Bearer <access_token>`
  - Время жизни токена составляет **60 минут** (`ACCESS_TOKEN_EXPIRE_MINUTES = 60`).

### Таблица ролей сотрудников
Система реализует ролевую модель (RBAC). Права доступа зависят от роли вошедшего сотрудника:
- **`admin`** — Администратор системы. Полный доступ ко всем операциям и журналу аудита.
- **`inspector`** — Инспектор ДПС. Оформление ДТП, розыск/снятие с розыска, поиск АТС (персональные данные владельцев маскируются).
- **`registrar`** — Регистратор. Регистрация транспортных средств, снятие с учета, просмотр реестра регистраций.
- **`to_employee`** — Сотрудник станции ТО. Внесение результатов техосмотра, добавление пунктов ТО, поиск АТС (персональные данные владельцев маскируются).
- **`analyst`** — Аналитик. Просмотр отчетов, графиков угонов, списка регистраций и общей статистики.

## 📝 3. Подробное описание эндпоинтов

### 3.1. Модуль авторизации и профиля

#### `POST /api/auth/login`
* **Описание:** Авторизация сотрудника и выдача JWT-токена.
* **Тип запроса:** `application/x-www-form-urlencoded`
* **Параметры формы (Form Data):**
  - `username` (string, обязательно): Табельный номер сотрудника (целое число, например, `1001`).
  - `password` (string, обязательно): Пароль сотрудника.
* **Успешный ответ (200 OK):**
  ```json
  {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "bearer",
    "id_employee": 1001,
    "role": "admin",
    "full_name": "Иванов Иван Иванович"
  }
  ```
* **Возможные ошибки:**
  - `400 Bad Request`: `"Табельный номер должен быть числом"`
  - `401 Unauthorized`: `"Неверный табельный номер или пароль"`

#### `GET /api/auth/me`
* **Описание:** Получение данных профиля текущего авторизованного сотрудника.
* **Заголовки:** `Authorization: Bearer <token>`
* **Успешный ответ (200 OK):**
  ```json
  {
    "id_employee": 1001,
    "last_name": "Иванов",
    "first_name": "Иван",
    "patronymic": "Иванович",
    "position": "Начальник отдела",
    "role": "admin"
  }
  ```

### 3.2. Универсальный поиск АТС (UC-01)

#### `GET /api/vehicles/search`
* **Описание:** Универсальный поиск ТС по частичному или полному совпадению: VIN, номеру кузова, номеру двигателя или государственному знаку (ГРЗ). 
* **Особенности:**
  - Алгоритм автоматически нормализует пробелы/дефисы в ГРЗ и заменяет латинские буквы-омоглифы на кириллицу (например, латинская `X` заменится на русскую `Х`).
  - Для ролей `inspector` и `to_employee` персональные данные владельца маскируются (`ФИО -> Иванов И.И.`, `Адрес -> Замаскировано...`, `Тел -> +7(9***`).
  - Факт поиска логируется в журнал аудита (включая результат поиска).
* **Параметры строки запроса (Query Params):**
  - `query` (string, обязательно): Поисковый запрос (например, `O-OO 777`, `VIN12345...`).
* **Успешный ответ (200 OK):**
  ```json
  {
    "id_vehicle": 5,
    "vin": "VIN88990011223344",
    "body_number": "BODY-8899",
    "engine_number": "ENG-5544",
    "chassis_number": "CHAS-1122",
    "brand_name": "Toyota",
    "model_name": "Camry",
    "color_name": "Белый",
    "year_produced": 2021,
    "engine_volume": 2.5,
    "registration_status": "Зарегистрировано",
    "grz_number": "О-ОО 777",
    "owner_name": "Петров П.П.",
    "owner_address": "Замаскировано (конфиденциальные данные)",
    "owner_phone": "+7(900***",
    "owner_type": "individual",
    "owner_inn": null,
    "is_stolen": false,
    "stolen_reason": null,
    "stolen_date": null,
    "to_history": [
      {
        "date_conducted": "2025-05-10",
        "date_next": "2026-05-10",
        "organization_name": "ООО Техосмотр-Запад"
      }
    ],
    "accident_history": [
      {
        "date_time": "2025-08-20T14:35:00",
        "location": "г. Москва, ул. Ленина, д. 45",
        "accident_type": "Столкновение",
        "description": "Несоблюдение дистанции водителем Camry",
        "damage_degree": "Легкая"
      }
    ]
  }
  ```
* **Возможные ошибки:**
  - `404 Not Found`: `"АТС не найдено в базе данных"`

* **Описание:** Регистрация транспортного средства на физическое или юридическое лицо с выдачей свободного ГРЗ.
* **Доступ:** `registrar`, `admin`
* **Тело запроса (JSON):**
  ```json
  {
    "vin": "VIN77788899900011",
    "body_number": "BODY-777",
    "engine_number": "ENG-999",
    "chassis_number": null,
    "id_model": 3,
    "id_color": 2,
    "year_produced": 2022,

    "owner_address": "г. Казань, ул. Мира, д. 12",
    "owner_phone": "+7(843)111-22-33",
    "owner_type": "legal",
    "owner_inn": "1655123456",
    "id_grz": 12
  }
  ```
* **Успешный ответ (200 OK):**
  ```json
  {
    "id_registration": 8,
    "id_vehicle": 15,
    "vin": "VIN77788899900011",
    "brand_name": "LADA",
    "model_name": "Granta",
    "color_name": "Серый",
    "grz": "А-АВ 116",
    "owner_name": "ООО Ромашка",
    "owner_address": "г. Казань, ул. Мира, д. 12",
    "owner_phone": "+7(843)111-22-33",
    "owner_type": "legal",
    "owner_inn": "1655123456",
    "date_registered": "2026-06-11",
    "is_deregistered": false,
    "date_deregistered": null
  }
  ```
* **Возможные ошибки:**
  - `400 Bad Request`: `"Регистрация заблокирована: АТС находится в розыске!"` (если VIN числится в активном розыске).
  - `400 Bad Request`: `"АТС с таким VIN уже имеет активную регистрацию (ГРЗ: ...)"`.
  - `400 Bad Request`: `"Выбранный ГРЗ занят или не существует"`.

#### `POST /api/vehicles/deregister`

  ```json
  }
  ```
* **Успешный ответ (200 OK):**
  ```json

    "id_vehicle": 15,
    "vin": "VIN77788899900011",
    "brand_name": "LADA",
    "model_name": "Granta",
    "color_name": "Серый",
    "grz": "А-АВ 116",
    "owner_name": "ООО Ромашка",
    "owner_address": "г. Казань, ул. Мира, д. 12",
    "owner_phone": "+7(843)111-22-33",
    "owner_type": "legal",
    "owner_inn": "1655123456",
    "date_registered": "2026-06-11",
    "is_deregistered": true,
    "date_deregistered": "2026-06-11"
  }
  ```
* **Возможные ошибки:**
  - `404 Not Found`: `"АТС с таким VIN не найдено"`.
  - `400 Bad Request`: `"У этого АТС нет активных регистраций"`.

#### `GET /api/registrations`
* **Описание:** Просмотр полного списка регистрационных действий в системе.
* **Доступ:** `registrar`, `analyst`, `admin`
* **Успешный ответ (200 OK):** Список регистрационных карточек (массив объектов `RegistrationResponse`).

## 🔍 4. Общие справочники (Endpoints)

Данные эндпоинты возвращают массивы сущностей для заполнения форм и фильтров в интерфейсе (селекторы моделей, цветов кузова, причин и т.д.):

- `GET /api/grz/free` — свободные государственные знаки (статус `"free"`).
- `GET /api/models` — доступные в базе модели ТС, включая связанные марки и производителей.
- `GET /api/colors` — список цветов.
- `GET /api/to-organizations` — аккредитованные организации техосмотра.
- `GET /api/accident-types` — виды ДТП (столкновение, наезд на пешехода, опрокидывание и др.).
- `GET /api/accident-reasons` — причины ДТП (превышение скорости, пьяное вождение, неисправность и др.).
- `GET /api/search-reasons` — основания для объявления в розыск (угон, скрытие с ДТП и др.).
