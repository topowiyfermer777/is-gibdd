# Руководство пользователя ИС ГИБДД

Добро пожаловать в Руководство пользователя **Информационной системы ГИБДД (ИС ГИБДД)**. Данное приложение разработано для автоматизации ключевых процессов сотрудников ГИБДД: поиска транспортных средств, регистрации, ведения розыска, техосмотра, учета ДТП и аналитики.

---

## 1. 🔑 Вход в систему и разграничение доступа

Вход в систему осуществляется на странице авторизации. Каждый сотрудник входит в приложение с помощью своего **табельного номера** и **пароля**.

### Тестовые учетные записи:
- **Администратор (`admin`):** Табельный номер `1001`, пароль `password`. Полный доступ ко всем вкладкам и журналу безопасности.
- **Инспектор ДПС (`inspector`):** Табельный номер `1002`, пароль `password`. Оформление ДТП, розыск ТС, проверка по базе (персональные данные владельцев маскируются).
- **Регистратор (`registrar`):** Табельный номер `1003`, пароль `password`. Постановка ТС на учет, снятие с учета.
- **Сотрудник ТО (`to_employee`):** Табельный номер `1004`, пароль `password`. Регистрация ТО и пунктов ТО.
- **Аналитик (`analyst`):** Табельный номер `1005`, пароль `password`. Просмотр дашбордов и аналитики.

## 2. 🔍 Модуль: Универсальный поиск АТС (UC-01)
Доступен всем ролям. Позволяет оперативно получить полную информацию об автомобиле.

### Как выполнить поиск:
1. Перейдите в раздел **«Поиск АТС»**.
2. Введите в поле поиска **Государственный номер (ГРЗ)**, **VIN-код**, **Номер кузова** или **Номер двигателя**.
   - *Примечание:* Поиск устойчив к раскладке клавиатуры. Если вы введете латинские буквы вместо русских в ГРЗ (например, `O-OO 777` вместо `О-ОО 777`), система автоматически сопоставит омоглифы и найдет ТС.
3. Нажмите кнопку **«Найти»**.

### Отображение результатов:
- **Общие сведения:** Марка, модель, цвет, год выпуска, объем двигателя, VIN.
- **Статус ТС:** 
  - Зеленый бейдж **«Зарегистрировано»** — автомобиль стоит на учете.
  - Серый бейдж **«Снято с учета»** — регистрация аннулирована.
- **⚠️ ТС В РОЗЫСКЕ!:** Если автомобиль угнан, карточка подсвечивается **красной рамкой** с мигающим баннером. В баннере указана дата объявления и причина. Для сотрудников на постах доступна кнопка **«Сообщить дежурному»** для мгновенной отправки тревожного сигнала.
- **Персональные данные (ПД):** ФИО, адрес, телефон и ИНН владельца. 
  - *Внимание:* Для ролей **Инспектор ДПС** и **Сотрудник ТО** данные владельца автоматически маскируются (скрываются символами `***`) для соответствия 152-ФЗ.
- **История ТО и ДТП:** Полный список пройденных осмотров и дорожно-транспортных происшествий, в которых участвовало данное ТС.

## 🚗 3. Модуль: Регистрация АТС и снятие с учета (UC-02)
Доступен ролям **Регистратор** и **Администратор**.

### Шаг 1: Сведения о владельце
1. Выберите тип владельца: **Физическое лицо** или **Юридическое лицо**.
2. Введите ФИО/название организации, адрес регистрации, телефон и ИНН (только для юрлиц).
3. Нажмите **«Далее»**.

### Шаг 2: Характеристики автомобиля
1. Введите **17-значный VIN-код**.
2. Выберите модель и цвет из предустановленных справочников.
3. Укажите год выпуска, объем двигателя, номера кузова, двигателя и шасси.
4. Нажмите **«Далее»**.

### Шаг 3: Выдача ГРЗ
1. Система автоматически подгрузит список свободных государственных знаков из базы данных.
2. Выберите нужный номерной знак из списка.
3. Нажмите **«Далее»**.

### Шаг 4: Подтверждение
1. Проверьте правильность введенных данных на сводной странице.
2. Нажмите кнопку **«Зарегистрировать»**.
3. При успешном завершении откроется **Электронное свидетельство о регистрации** с возможностью распечатки.

### Снятие ТС с учета:
1. Перейдите во вкладку **«Снятие с учета»**.
2. Введите 17-значный VIN-код ТС и нажмите **«Снять с учета»**.
3. Гознак автомобиля вернется в список свободных номеров, а ТС перейдет в архивный статус.

## 🚨 4. Модуль: Реестр розыска транспортных средств (UC-03)
Доступен для просмотра всем пользователям. Вносить изменения могут только **Инспектор ДПС** и **Администратор**.

### Как объявить автомобиль в розыск:
1. Перейдите в раздел **«Розыск ТС»**.
2. Введите VIN-код ТС.
3. Выберите причину розыска (Угон, Скрытие с места ДТП, Залог, Иное).
4. Укажите подробные обстоятельства в поле «Детали».
5. Нажмите кнопку **«Объявить розыск»**. Автомобиль мгновенно перейдет в статус розыска на всех постах.

### Как снять автомобиль с розыска:
В таблице активных розысков найдите нужное ТС и нажмите кнопку **«Найдено»** в колонке «Действия».

---

## 🔧 5. Модуль: Технический осмотр (UC-04)
Доступен для просмотра всем пользователям. Вносить записи могут **Сотрудник ТО** и **Администратор**.

### Внесение результатов ТО:
1. Перейдите во вкладку **«Внести результаты ТО»**.
2. Введите VIN-код автомобиля.
3. Выберите вашу аккредитованную организацию из списка.
4. Укажите дату проведения осмотра и рекомендуемую следующую дату.
5. Нажмите **«Зафиксировать ТО»**.

### Добавление новой организации ТО:
1. Во вкладке **«Пункты техосмотра»** заполните форму: Наименование, Адрес, Телефон, Email.
2. Нажмите **«Добавить в реестр»**.

---

## 💥 6. Модуль: Учет дорожно-транспортных происшествий (ДТП)
Доступен для просмотра всем пользователям. Оформлять протоколы могут **Инспектор ДПС** и **Администратор**.

### Оформление протокола ДТП:
1. Перейдите во вкладку **«Оформить протокол»**.
2. Укажите дату/время, место аварии, тип и основную причину происшествия.
3. Укажите количество пострадавших/погибших и погодные условия.
4. **Участники ДТП:** Нажмите кнопку **«Добавить участника»** для каждого поврежденного ТС.
   - Введите VIN-код каждого автомобиля-участника.
   - Выберите степень повреждения кузова (Легкая, Средняя, Тяжелая, Тотал).
   - Укажите оценочную сумму ущерба в рублях.
5. Заполните текстовое описание обстоятельств аварии.
6. Нажмите **«Оформить ДТП»**. Запись будет сохранена в журнале.

---

## 📊 7. Аналитика и Системный аудит

### Аналитический дашборд (роли `analyst`, `admin`):
- Перейдите в раздел **«Аналитика»** для просмотра ключевых показателей: общего количества авто в базе, активных розысков и процента раскрываемости угонов.
- Интерактивные индикаторы наглядно показывают распределение угонов по маркам автомобилей.

### Журнал безопасности (только роль `admin`):
- Администратор может зайти в раздел **«Аудит»** для контроля логов безопасности.
- В журнале фиксируются все действия: авторизации сотрудников, поисковые запросы по VIN/ГРЗ, операции регистрации, розыска и оформления ДТП.
- Доступна быстрая фильтрация по ролям, типам действий и текстовый поиск по деталям лога.
