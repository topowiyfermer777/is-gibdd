# Руководство системного администратора ИС ГИБДД

Данный документ предназначен для системных администраторов ИС ГИБДД и описывает архитектуру развертывания, методы сопровождения, резервного копирования и мониторинга системы.

---

## 1. 🏗️ Архитектура развертывания

Система спроектирована как веб-приложение на базе микросервисной архитектуры и разворачивается с помощью **Docker Compose** в изолированной сети:

```text
               +-----------------------------------------+
               |             Хост-компьютер              |
               |                                         |
               |  +------------------+                   |
               |  |  gibdd-frontend  |                   |
               |  |  (Nginx, р: 80)  | <---- порт 5177   |
               |  +--------+---------+                   |
               |           |                             |
               |    проксирование /api                   |
               |           |                             |
               |           v                             |
               |  +------------------+                   |
               |  |  gibdd-backend   |                   |
               |  | (FastAPI, р:8080)| <---- порт 8080   |
               |  +--------+---------+                   |
               |           |                             |
               |   запись и чтение данных                |
               |           v                             |
               |  +------------------+                   |
               |  |  Том db-data     |                   |
               |  | (/app/db/...)    |                   |
               |  +------------------+                   |
               +-----------------------------------------+
```

1. **gibdd-frontend (Порт 5177):** Контейнер на базе веб-сервера Nginx, который раздает скомпилированное React SPA приложение и перенаправляет все входящие запросы с префиксом `/api` на бэкенд.
2. **gibdd-backend (Порт 8080):** Серверное FastAPI-приложение, обрабатывающее бизнес-логику и осуществляющее транзакции к базе данных SQLite.
3. **Volume db-data:** Физический том Docker, монтируемый в контейнер бэкенда по пути `/app/db/`. В нем хранится файл базы данных `gibdd.db` для обеспечения сохранности данных при перезапусках.

## 🛠️ 2. Установка и запуск системы

### Предварительные требования:
На сервере/компьютере должен быть установлен **Docker Desktop** (для Windows/macOS) или **Docker + Docker Compose CLI** (для Linux).

### Команды управления контейнерами:
Все команды выполняются из папки проекта `is/` (где лежит файл `docker-compose.yml`):

- **Запуск системы в фоновом режиме (рекомендуется):**
  ```bash
  docker compose up --build -d
  ```
- **Просмотр состояния запущенных контейнеров:**
  ```bash
  docker compose ps
  ```
- **Просмотр системных логов бэкенда и Nginx:**
  ```bash
  docker compose logs -f
  ```
- **Остановка контейнеров с сохранением данных:**
  ```bash
  docker compose down
  ```
- **Остановка контейнеров с полным удалением тома базы данных (полный сброс):**
  ```bash
  docker compose down -v
  ```

## 💾 3. Резервное копирование и восстановление базы данных

База данных ИС ГИБДД хранится в едином файле SQLite. Внутри контейнера `backend` путь к базе данных: `/app/db/gibdd.db`.

### Создание резервной копии:
Поскольку SQLite блокирует базу данных при записи, для создания корректного бэкапа "на лету" (без остановки сервера) используется встроенный механизм `.backup` утилиты `sqlite3`.

Выполните команду на хост-компьютере для создания резервной копии базы данных в текущую папку:
```bash
docker exec -t gibdd-backend sqlite3 /app/db/gibdd.db ".backup '/app/db/backup.db'"
docker cp gibdd-backend:/app/db/backup.db ./gibdd_backup_$(date +%F).db
docker exec -t gibdd-backend rm /app/db/backup.db
```

### Настройка автоматического резервного копирования (для Linux):
Добавьте задачу в планировщик `cron` (выполнять каждые 6 часов):
```cron
0 */6 * * * docker exec -t gibdd-backend sqlite3 /app/db/gibdd.db ".backup '/app/db/backup.db'" && docker cp gibdd-backend:/app/db/backup.db /var/backups/gibdd/gibdd_backup_$(date +\%F_\%H\%M).db && docker exec -t gibdd-backend rm /app/db/backup.db
```

### Восстановление базы данных из бэкапа:
1. Остановите контейнеры: `docker compose down`.
2. Скопируйте файл бэкапа поверх старого файла в каталоге тома Docker. Путь к томам на хост-системе зависит от ОС (на Linux по умолчанию: `/var/lib/docker/volumes/is_db-data/_data/gibdd.db`).
3. Запустите контейнеры: `docker compose up -d`.

## 🚨 4. Системный аудит и безопасность

Все изменения важных сущностей (поиск, регистрация, угон, ДТП, ТО) логируются в таблицу `audit_logs` базы данных.

### Просмотр логов безопасности:
1. Администратор может просматривать логи через веб-интерфейс в разделе **«Аудит»**.
2. В случае необходимости выгрузить аудит-лог напрямую из SQLite в формате CSV, выполните:
   ```bash
   docker exec -it gibdd-backend sqlite3 -header -csv /app/db/gibdd.db "SELECT * FROM audit_logs;" > audit_export.csv
   ```
